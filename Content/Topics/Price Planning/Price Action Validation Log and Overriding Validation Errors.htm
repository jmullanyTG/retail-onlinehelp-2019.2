Price Action Validation Icons, the Validation Log, and Overriding Validation Errors

The system returns messages indicating if a Price Action has successfully complied with Validation rules. Validation Severity failure icons on the Price Actions grid provide convenient indicators of whether a Price Action is validated or not while.

To display the Validation Log, double-click the Validation Severity failure icon in the Validation field on the Price Actions, Variant Summary, Items, or Item Location grids.

The Validation Log provides messages based on failed validations configured in the Price Strategy screen. Depending on where you display the Validation Log, the results are filtered as appropriate. The Validation Log can viewed from any Price Actions grid (Price Action Header, Summary, Item and Item Location). Simply double-click on the icon in the Validation column in the Price Actions grid.

NOTE: A failed validation that has been reviewed is now considered no longer failed and will be considered for export. Also, not all Validations can be overridden. The ability to override is based on system configuration which in turn is based on a review of the customer’s business process.

Validation Log

The Validation Log has advanced filtering that lets users review and mark as reviewed validation failure records allowing Promotions to continue to downstream systems. The Validation Log does not limit the number of records to be returned.

Product Validation failures are filtered based on where the Validation log originated:

Level Filter
Price Action All Site Cluster Validation failures, at the Inventory level, for ALL Site Clusters
Site Cluster All Variant's Item Master's Inventory Validation failures for the specific Site Cluster
Variant All associated Item Masters' Inventory Validation failures for the specific Site Cluster
Item Master All associated Inventory Validation failures for the specific Site Cluster
Inventory Validation failures for the specific Site Cluster

Evaluating Pricing Validation failures at a Variant and Site level

You can reduce the number of records that need evaluating by electing to evaluate Pricing Validation failures at a Variant and Site level. To enable this functionality, system policy Advanced Validation Log needs to be set to True.

Overriding a Validation failure

The ability to override Validation failures reduces the amount of time Merchant Managers spend reviewing Validation failures. To override a Validation failure, select the Override check box and click Ok.

NOTE: Overriding a Validation failure causes the system to:

  • update any higher level affected by the Price Action’s worst case Validation Severity up to the Site Cluster, Price Action and Plan levels (these levels will still display worst case aggregation, meaning that while an override might be made, if there is another ‘worst’ case (error) the higher levels will not be updated):
    • If the override is at the Inventory level, then the system will update the Item Manager, Variant, Site Cluster, Plan, and Action Header level, if appropriate.
    • If the override is at the Item Master level, then the system will update the Variant, Site Cluster, Plan, and Action Header level, if appropriate.
    • If the override is at the Variant level, then the system will update the Site Cluster, Plan, and Action Header level, if appropriate.

Exporting Price Actions that have a Validation Severity type of Error

The system does not allow Price Actions with a failed Validation Severity type of Error to be exported. Price Actions with Validation failures of severity type Error are exportable to downstream systems (such as reporting, ERP, POS) when all Validation failures of severity type Error have been overridden.

The following validation states can be exported:

Order of Validation Severity levels

The order of the Validation Severity levels (from best to worst) is as follows:

  • Validated
  • Reviewed
    • Information Overridden
    • Warning Overridden
    • Error Overridden
  • Information Provided
  • Warning Found
  • Error Encountered

Rules governing the recalculation and revalidation of a Price Action

When recalculating and revalidating a Price Action, the system will:

  • keep the Validation severity state when the Validation failure matches the previous failure (an Error remains an Error, a Warning remains a Warning, and so on).
  • update the Validation severity state if the Validation failure does not match the previous failure (an Error becomes a Warning, A Warning becomes an Error, and so on).
  • discard the Validation severity state if the Validation no longer fails (an Error is now Validated, a Warning is now Validated, and so on).
