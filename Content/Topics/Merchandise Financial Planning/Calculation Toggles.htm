Calculation Toggles

The Calculation Toggles pane defines which dependent metrics recalculate and which ones hold when changes are made.

If a user changes a week's Markdown, then that week's COGS will hold (but the following weeks to do not). If the user changes a month's Markdown, it spreads the change across the weeks. The first week in the month's COGS stays the same, but because the Markdowns changed, the first week's Sales Units and Total Original Retail change, which in turn changes EOP Units, EOP Cost, EOP Retail, and Markdown Cost. Because the first week's EOPs changed, the second week's BOPs changed.

COGS is based on the ratio between BOP Cost and BOP Retail, so it necessarily changes in the second week. Similarly in the third through nth weeks. Thus, the total COGS for the month changes.

  • Calculation toggles do not require saving to be in effect. The change is immediate once the selection is made.

  • When alternate (non-default) calculation toggles are selected, they appear in darker gray for easy identification at a glance
