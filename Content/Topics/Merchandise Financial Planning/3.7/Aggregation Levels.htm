Aggregation Levels

In order to provide a more fluid user experience, the Parameters pane includes product and channel aggregation levels. Data for the selected portions of the product and site hierarchies will be pre-aggregated to these levels in temporary database tables, thus reducing the total number of rows that must be read/written and improving overall system performance.

Aggregation levels can be changed at any time without consequence to the work users have done. Data will simply be re-aggregated/disaggregated. Note, however, that actual values are always stored at the lowest plan levels in the database, regardless of what levels you are currently aggregated to. Changes made at higher levels are simply broken down by percent contribution of the previous values.

  • Example:based on the aggregation level selected, the user will see values for that level as well as those above.
