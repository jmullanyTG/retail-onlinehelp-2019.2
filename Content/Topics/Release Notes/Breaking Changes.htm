Breaking Changes

Assortment Planning DMDEV-56319

On Order is now cached and updated asynchronously. Note the following:

  • Customization referencing any of the following changed objects needs to be updated to reflect the new paradigm:

    • assortment.Engine_ReadXML

    • assortment.LineViz_Placeholder_Select

    • assortment.LineViz_PlaceholderCollection_Select

    • assortment.LineViz_PlaceholderInstance_EOPBOP_Calculate

    • assortment.LineViz_View_Cluster_Select

    • assortment.Maintenance_Actualize

    • assortment.Maintenance_SalesPlan_Recalculate

    • assortment.Maintenance_StockOnHand_Actualize

    • assortment.ReceiptPlan_UpdatePost

    • assortment.Report_BuySheet_Select

    • assortment.Report_Rollup_Select

    • assortment.SalesReceiptPlan_Select

  • The procedure assortment.SplitOutstandingQuantity_Update is deprecated; all references in customization should be removed.
  • Customization inserting, updating, or deleting on tbl_SupplierOrderLine, tbl_SupplierOrderLineVariant, and orderamanger.Order_Line_Commitment_Variant needs to be updated to call the pubsub.Publish procedure, examples of which can be found in
    • assortment.Commitment_Generate
    • dbo.usp_API_Import_SupplierOrder_Line_Variant_Post
    • dbo.usp_API_Import_SupplierOrder_Line_Post_ByPartition
  • The receipts line shown in Receipt Plan used to be inclusive of order information, but now it is additive.

Merchandise Financial Planning DMDEV-59298

There is a new parameter in the CUSP hooks called Currency Neutralization. Any customized stored procedures for InitialSelect will need to be modified to be functional.

Price Manager

A new nightly job labeled Pricing Plan and Action Cleanup cleans up historical Pricing Plans and Actions to maintain database growth. This new nightly job needs to be configured to the customer data retention policies.
